Fundanet Multitenant

Sistema central de servicios compartidos para todo el ecosistema Fundanet

Descripción del Proyecto

Fundanet Multitenant está compuesta por dos sistemas independientes: una API que expone servicios y una web que permite realizar operaciones administrativas. Su misión es exponer servicios comunes a todas las aplicaciones del ecosistema Fundanet.

API de Servicios

Expone servicios comunes para gestión de configuraciones, traducciones, usuarios, permisos, roles, tenants, trazabilidad, despliegue, licenciamiento y métricas de uso.

Web Administrativa

Aplicación en Blazor WebAssembly para configurar usuarios, licencias, módulos de acceso y sistema de despliegue. Exclusiva para administradores de Semicrol.

Arquitectura y Flujo de Comunicación

Flujo: Las aplicaciones cliente se conectan a Azure API Management, que actúa como proxy seguro hacia la API Multitenant. La API gestiona servicios comunes y persiste datos en Azure SQL Database. La Web Admin permite la configuración del sistema mediante autenticación B2C.

Ventajas del Sistema

Reutilización de Componentes

Todas las aplicaciones consumen servicios comunes sin duplicar lógica de negocio (usuarios, permisos, licencias, configuración, trazabilidad)

Centralización y Mantenimiento

Al centralizar servicios en una API, se evita replicar funcionalidades y se reduce el mantenimiento

Escalabilidad Independiente

La API puede escalar horizontalmente de manera independiente a la interfaz web administrativa

Seguridad y Rendimiento

Azure API Management actúa como capa de orquestación, seguridad y caché, mejorando disponibilidad, control de acceso y rendimiento

Evolución y Roadmap

Integraciones

  • • Sincronización con Azure B2C

Fundanet Message System

Sistema de interoperabilidad asíncrona basado en Event-Driven Architecture

Descripción del Proyecto

Sistema que permite la interoperabilidad entre aplicaciones de forma asíncrona mediante Event-Driven Architecture (EDA). Las aplicaciones o microservicios se comunican mediante eventos o mensajes, en lugar de llamadas directas, desacoplando los servicios y mejorando la escalabilidad.

Integración por API

Los clientes solicitan periódicamente los mensajes pendientes mediante polling

Integración Reactiva (Webhooks)

Cada acción comunica directamente a las APIs de los clientes en tiempo real

Arquitectura y Flujo de Comunicación

Flujo: 1) Los eventos se registran en Outbox (transacción local), 2) Se publican en Azure Service Bus, 3) Worker Services consumen los mensajes, 4) Se validan en Inbox (idempotencia), 5) Se integran en aplicaciones destino mediante API REST o Webhooks.

Arquitectura y Patrones

Event-Driven Architecture (EDA)

Comunicación mediante eventos en lugar de llamadas directas, mejorando escalabilidad y desacoplamiento

Patrón Outbox

Garantiza consistencia entre base de datos y mensajes enviados, registrando eventos en tabla local

Patrón Inbox

Asegura idempotencia evitando procesar dos veces el mismo mensaje mediante registro de consumidos

Procesamiento Asíncrono

Worker Services y Azure Functions procesan mensajes de forma asíncrona y resiliente

Tecnologías Utilizadas

.NET Core 9

Azure Service Bus

Message broker para eventos JSON/Protobuf

Worker Services

Procesos background para consumo de mensajes

Azure Functions

Endpoints de integración para clientes

Protocolo binario para mensajes seguros

Beneficios Clave

Desacoplamiento Total

Tolerancia a Fallos

Sistema resiliente ante cortes de Internet, con reintentos controlados y recuperación automática

Fundanet AI Tickets

Sistema de gestión inteligente de tickets y gastos

Descripción

Fundanet.Tickets es una solución desarrollada por Semicrol para la digitalización y automatización del proceso de gestión de tickets y justificantes de gasto. Permite a los usuarios capturar tickets o facturas desde sus dispositivos móviles mediante una fotografía, que el sistema analiza automáticamente con IA para extraer los datos relevantes (importe, fecha, proveedor, IVA, etc.) antes de su validación e integración con las aplicaciones corporativas de Fundanet (Corporate, CTMS, IEC, etc.).

Implementada en .NET Core 9, que ofrece servicios REST para el almacenamiento, procesamiento y validación de tickets, así como la integración con los sistemas de Fundanet. Se despliega detrás de Azure API Management, garantizando balanceo de carga, seguridad, trazabilidad y escalado dinámico.

Para usuarios administradores y corporativos. Permite revisar los tickets procesados, validar los datos extraídos por IA y aprobar su integración con los sistemas de Fundanet.

Aplicaciones Móviles (Flutter)

Aplicaciones móviles multiplataforma (Flutter) para Android y iOS, que permiten subir tickets mediante captura de imagen. Estas aplicaciones se comunican con la API para realizar el reconocimiento OCR, validar la información y registrar los datos procesados.

Fundanet Nexus

Plataforma de interconectividad de sistemas en Centros de Investigación

Descripción

Nexus es una plataforma integral diseñada para interconectar diferentes sistemas y plataformas utilizados en los Centros de Investigación facilitando la comunicación y garantizando una integración fluida.

Su principal objetivo es facilitar la interoperabilidad entre aplicaciones, historiales clínicos, sistemas de análisis y dispositivos de monitorización, promoviendo una gestión eficiente de la información y mejorando la colaboración entre herramientas especializadas.

Arquitectura y Flujo de Comunicación

Flujo: Nexus actúa como hub central de interoperabilidad conectando múltiples sistemas (Fundanet CTMS/iPharma, SAP Argos, OpenClinica, RedCap, E-CAP, SISCat, Fenix ERP) y dispositivos (Apple Health, Google Fit) mediante Mirth Connect, .NET Core 9 y Azure Service Bus usando protocolos estándar (HL7 FHIR, OMOP CDM, API REST).

Fundanet Suite

Lanzador unificado de aplicaciones con autenticación centralizada

Descripción

La Suite de aplicaciones integra un sistema de Autenticación único basado en Azure B2C, de forma que cada usuario que acceda una vez autenticado tenga la opción de lanzar cualquier aplicación de la Suite a la que se tiene acceso.

Una suite de aplicaciones es un conjunto de programas o aplicaciones de software que están diseñados para trabajar juntos y cumplir con una serie de funciones relacionadas, generalmente dentro de un mismo entorno o para un propósito común. Estas aplicaciones suelen estar integradas de manera que puedan compartir datos y funcionalidades, facilitando la colaboración y optimizando los flujos de trabajo.

La idea es tener una entrada común para cada usuario, de forma que este pueda acceder a cada uno de los programas. Esto conforma un punto de entrada único.

Arquitectura y Flujo

Flujo: Usuario accede → Azure B2C autentica con OAuth/SSO → Suite Launcher (Blazor WASM) valida licencias y permisos con Multitenant API → Muestra dashboard con estadísticas, notificaciones y caducidad de licencias → Usuario lanza aplicaciones disponibles (Corporate, CTMS, IEC, etc.).

Características

Punto de Entrada Único

Una entrada común para cada usuario, de forma que este pueda acceder a cada uno de los programas de la suite

Autenticación OAuth con Azure B2C

Sistema de autenticación único basado en B2C, Single Sign-On para todas las aplicaciones

Dashboard Integrado

El sistema además puede mostrar estadísticas y notificaciones, caducidad de las licencias, incidencias en las aplicaciones, estadísticas de uso, etc.

Validación Centralizada

Valida los usuarios, las licencias y las aplicaciones disponibles y las presenta para poder acceder a ellas con facilidad

Tecnología

Blazor WebAssembly

Aplicación SPA (Single Page Application) desarrollada en Blazor Web Assembly que usa una autenticación OAuth con Azure B2C

Ventajas

Experiencia unificada: un solo punto de acceso para todas las aplicaciones del ecosistema

Seguridad centralizada: gestión de autenticación y autorización desde un único sistema

Visibilidad completa: dashboard con información relevante sobre licencias, notificaciones e incidencias

Facilita la colaboración: usuarios pueden moverse fluidamente entre aplicaciones sin re-autenticarse

### Fundanet Multitenant. 

Descripción: Está compuesta por dos sistemas independientes, una API que expone servicios y una web que permite realizar ciertas operaciones. Su misión es exponer servicios comunes a todas las aplicaciones. Esta solución está compuesta por dos elementos principales, la propia API y una Web de Administración.

- La API expone servicios a distintas aplicaciones como Fundanet Corporate,CTMS, IEC, Arukon, Fundraising, etc. Esta permite usar servicios comunes para la gestión de las distintas configuraciones, traducciones, usuarios, permisos y roles, Tenants, Trazabilidad, Despliegue, Licenciamiento, Gestión por anillos, Métricas de uso, etc.). Para mejorar el escalado, la seguridad y el consume, esta se explota a través de un servicio llamado Azure Api Management.
- Una aplicación Web en Blazor WebAssembly que permite configurar usuarios, licencias, módulos de acceso, sistema de despliegue, etc. Esta Web solo es usada por los usuarios administradores de Semicrol. 
Tecnologías utilizadas:
- Net Core 9
- Asp Aspire
- Azure Sql Database. Usada para persitencia de usuarios, tenants, configuraciones, accesos, roles, etc.
- Blazor Web Assembly
- Soporta versionado
- Healthy checks
- Integración con B2C para la administración
- Sistema de logs con Azure Insights
- Azure Monitor para monitorización de uso.
- Azure App Service para la Api
- Azure App Service para la Web
- Azure API Management para mayor seguridad, balanceo de carga, etc. Las aplicaciones se conectan a este servicio que a su vez se conecta con la Api.
Ventajas:
- Las aplicaciones actuales como Fundanet Corporate, CTMS, IEC, Arukon, Fundraising, etc, consumen los servicios de la API sin duplicar lógica de negocio.
- El modelo favorece la reutilización de componentes y servicios comunes (usuarios, permisos, licencias, configuración, trazabilidad, etc.) para todo el ecosistema.
- Al centralizar los servicios en una API, se evita replicar funcionalidades y se reduce el mantenimiento.
- La API puede escalar horizontalmente de manera independiente a la interfaz web administrativa.
- Azure API Management actúa como capa de orquestación, seguridad y caché, mejorando la disponibilidad, el control de acceso y el rendimiento.
Evolución: El sistema continuará evolucionando para integrar sistemas comunes, gestor de informes, gestor documental, auditorias, trazabilidad, métricas de uso y todo tipo de servicios comunes a las aplicaciones. Actualmente se está trabajando por unificar las métricas de uso, la sincronización con B2B para integrar seguridad Oauth, el sistema de logs, etc.

Información adicional: Todas las aplicaciones usarán esta Api por lo que es crítica, una funcionalidad transversal bastante común es el sistema de traducciones que permite cambiar cualquier texto sin necesidad de desplegar, el sistema de despliegue continuo sobre base de datos, las gestión de las licencias de usuarios, etc.

### Sistema de mensajería 
Es un sistema que permite la interoperabilidad entre aplicaciones de forma asíncrona. Está basado en Event-Driven Architecture (EDA), que es un estilo de arquitectura en el que las aplicaciones o microservicios se comunican mediante eventos o mensajes, en lugar de llamadas directas (HTTP o RPC). Esto desacopla a los servicios, mejora la escalabilidad y permite procesar operaciones de forma asíncrona y garantizar las transacciones distribuidas entre distintas aplicaciones.
Usa colas de eventos en Azure Service Bus, integra patrones Outbox e Inbox para garantizar las transacciones y Services Workers o Azure function para integración con distintas aplicaciones. Se han diseñado conectores específicos para Corporate, Fundanet Legacy, CTMS y poco a poco se irán integrando el resto de las aplicaciones.
Es clave para el funcionamiento de Nexus ya que son sistemas asíncronos que necesitan garantizar la entrega. 
El sistema es completamente asíncronos diseñado para soportar fallos, escalable, capaz de procesar cientos de miles de mensajes en poco tiempo con la única limitación de las capacidades de los clientes, este sistema permite integración entre aplicaciones propias o externas a través de dos mecanismos:
	- La integración de mensajes a través de una API. Los clientes solicitan cada cierto tiempo los mensajes pendientes.
	- La integración de mensajes de forma reactiva a través de Webhooks, cada acción que se produce comunica directamente a las APIs de los clientes para que estos integren los cambios
Tecnologías utilizadas:
- Servicios de procesamientos de mensajes con Net Core 9
- Azure Service Bus: Message broker (bus de mensajería) que actúa como middleware para enviar y recibir eventos JSON entre servicios. 
- Se basa en mensajes JSON: Estos mensajes representan los eventos de dominio (por ejemplo, UsuarioCreado, PagoConfirmado, etc.) intercambiados entre servicios.
- Api Rest para exponer los endpoints de integración
- Utiliza Worker Services (.NET): Procesos de background que consumen los mensajes desde Service Bus e interaccionan con las aplicaciones para integrar la lógica de negocio 
- Utiliza Azure Functions para la el procesamiento de mensajes en la nube y su integración con Webhookds de clientes externos.
- Implementa el Patrón Outbox e Inbox: Que permite garantizar consistencia entre base de datos y mensajes enviados, registrando los eventos en una tabla “Outbox” y publicándolos de forma segura y asegurar idempotencia (Inbox, no procesar dos veces el mismo mensaje), guardando un registro de los mensajes ya consumidos.
- Aplicaciones para monitorizar y reintentar mensajes fallidos.
- Sistema de trazabilidad y auditoria 
- Resiliente ante fallos y cortes de Internet.
Beneficios clave:
- Desacopla los servicios productores y consumidores.
- Permite tolerancia a fallos y reintentos controlados.
- Asegura consistencia eventual entre bases de datos distribuidas.
- Facilita la escalabilidad horizontal y la resiliencia.
- Permite auditar los eventos y reproducirlos si es necesario.

Importante: Tanto el Api Multitenant como el sistema de mensajería permiten el diseño de una Suite de aplicaciones capaz de comunicarse entre sí de forma bidireccional, que comparten servicios comunes, pueden escalar en base a la demanda sin requerir más Infraestructura. Son capaces de soportar cientos e incluso miles de clientes con cientos de aplicaciones. Gracias a la infraestructura de la nube y el uso de Azure Service Bus soporta balanceo de carga, alta disponibilidad y seguridad usando mensajes en el protocolo binario protobuff.
De la misma forma, esta tecnología habilita la posibilidad de enlazar nuestras aplicaciones con aplicaciones de terceros y escalar de forma eficiente.

### Fundanet Web - Nueva arquitectura – iFundanet y nuevos productos

La nueva arquitectura web permitirá desarrollar aplicaciones escalables y mantenibles para Semicrol, usa una aproximación a Clean Architecture y un modelo basado en CQRS.

Arquitectura:

- Basada en arquitectura limpia (Clean Architecture)
- Soporte Multitenant, permite aislar datos y configuraciones por cliente o tenant, asegurando independencia y escalabilidad para entornos multiempresa.
- Separa de forma estricta las capas de dominio, aplicación, infraestructura y presentación. Esto garantiza independencia del framework y facilita el mantenimiento y las pruebas.
- Soporta DDD (Domain-Driven Design), aplica patrones como Value Objects, Agregados y Entidades Raíz, permitiendo que el modelo represente fielmente el dominio de negocio y sea extensible sin romper dependencias.
- Implementa comandos y consultas mediante CQRS. Separa la lectura (queries) de la escritura (commands), mejorando la mantenibilidad, la escalabilidad y la capacidad de auditoría.

Tecnologías:

- Implementada en net Core 9. aprovecha las últimas mejoras en rendimiento, soporte nativo de contenedores, asíncronos y seguridad del ecosistema .NET moderno.
- Uso de Entity Framework Core. Simplifica el acceso a datos mediante ORM, soportando migraciones automáticas, LINQ y compatibilidad con SQL Server Azure.
- Minimal API. Ofrece un modelo ligero, rápido y de bajo overhead para la exposición de endpoints HTTP, reduciendo complejidad y aumentando el rendimiento.
- Fluent Validation. Proporciona validaciones declarativas y reutilizables sobre los modelos de dominio y DTOs, asegurando coherencia y reduciendo errores de validación manual.
- Pruebas unitarias y de integración
- Asegura calidad y confiabilidad del c??digo mediante test automáticos que validan tanto la lógica interna como la interacción entre componentes.
- Seguridad basada en OpenID Connect
- Permite autenticación federada y autorización moderna, compatible con OAuth2, tokens JWT y escenarios B2B/B2C.
- Integración con Azure B2C
- Ofrece autenticación de usuarios externos, registro y gestión de identidades, integrando el sistema con el ecosistema de identidad de Azure.
- Como cliente usa Blazor WebAssembly que ejecuta C# directamente en el navegador, reduciendo la necesidad de JavaScript y aprovechando el mismo modelo de desarrollo que el servidor. Aunque actualmente utiliza esta tecnología, la arquitectura modular permite reemplazar o complementar la interfaz con otros frameworks front-end, facilitando su evolución y adopción en diferentes proyectos o clientes. Podría integrar otras tecnologías como React, Vue, Svelte.
- Utiliza la librería de componentes Arco basados en controles de Synfusion
Beneficios:

- Modularidad y mantenibilidad: cada capa tiene responsabilidades claras, lo que simplifica la evolución del sistema. 
- Escalabilidad y rendimiento: CQRS, Minimal API y la asincronía permiten procesar grandes volúmenes de peticiones de forma eficiente.
- Seguridad centralizada: integración nativa con OpenID Connect y Azure B2C.
- Extensibilidad futura: soporte multitenant y arquitectura limpia permiten incorporar nuevos módulos o microservicios fácilmente.
- Consistencia tecnológica: mismo lenguaje (C#) y ecosistema (.NET) en servidor y cliente, reduciendo la fricción entre capas.
- Calidad garantizada: validación, testing y diseño por dominio aseguran estabilidad y evolución sostenible.

Esta arquitectura establece una plataforma sólida para servicios de Fundanet Corporate, lista para integrar antiguos y nuevos servicios

Arquitectura:
- Arquitectura basada en CQRS y Minimal API
- Soporta Multitenant

Tecnologías:

- Implementada en net Core 9
- Seguridad basada en OpenIdConnect
- Implementación a través de Azure Api Management
- Asíncrona
- Soporta versionado
- Soporta trazabilidad y métricas de uso
- Gestión de Usuarios, roles y permisos

Beneficios clave:

- Escalabilidad y rendimiento: La API está diseñada para escalar tanto horizontal como verticalmente, permitiendo aumentar capacidad según la demanda.
- Al estar detrás de Azure API Management, puede balancear carga, cachear respuestas y limitar el tráfico por suscripción o aplicación.
- La arquitectura asíncrona y la separación CQRS optimizan el rendimiento en operaciones de lectura y escritura.
- Gracias al uso de .NET 9, la API aprovecha las mejoras en rendimiento, gestión de memoria y seguridad del framework.

- Permite auditar cada llamada, mejorando trazabilidad y cumplimiento normativo.
- Multitenancy y gobernanza, soporte multitenant nativo: múltiples organizaciones pueden usar la misma API de forma aislada y segura. Mayor escalabilidad a un mínimo coste.
- Simplifica la gestión de usuarios, roles y permisos, ofreciendo un modelo de acceso unificado y administrable desde un punto central.
- Facilita mantener coherencia entre entornos (Corporate, CTMS, IEC, etc.).

- Soporte de versionado de API que permite evolucionar servicios sin romper compatibilidad con los clientes existentes.

### Fundanet.Tickets – Sistema de gestión inteligente de tickets y gastos

Se trata de una solución desarrollada por Semicrol para la digitalización y automatización del proceso de gestión de tickets y justificantes de gasto. Permite a los usuarios capturar tickets o facturas desde sus dispositivos móviles mediante una fotografía, que el sistema analiza automáticamente con IA para extraer los datos relevantes (importe, fecha, proveedor, IVA, etc.) antes de su validación e integración con las aplicaciones corporativas de Fundanet (Corporate, CTMS, IEC, etc.).

La solución está compuesta por tres elementos principales:

- API central implementada en .NET Core 9, que ofrece servicios REST para el almacenamiento, procesamiento y validación de tickets, así como la integración con los sistemas de Fundanet.
Se despliega detrás de Azure API Management, garantizando balanceo de carga, seguridad, trazabilidad y escalado dinámico.

- Aplicación Web (Blazor WebAssembly) para usuarios administradores y corporativos.
Permite revisar los tickets procesados, validar los datos extraídos por IA y aprobar su integración con los sistemas de Fundanet.

- Aplicaciones móviles multiplataforma (Flutter) para Android y iOS, que permiten subir tickets mediante captura de imagen.

   Estas apliaciones se comunican con la API para realizar el reconocimiento OCR, validar la información y registrar los datos procesados.

Tecnologías utilizadas:

- NET Core 9 y ASP.NET Minimal API
- Blazor WebAssembly
- Flutter (aplicaciones móviles Android / iOS)
- Azure API Management (proxy inverso, seguridad, control de acceso y escalado)
- Microsoft AI Foundry con modelos de reconocimiento OCR e IA generativa
- Azure Blob Storage para almacenamiento seguro de las imágenes capturadas de tickets
- Azure SQL Database para almacenar los datos identificados por el OCR en formato JSON estructurado
- Seguridad basada en OpenID Connect
- Integración con Azure B2C para autenticación y gestión de usuarios
- Arquitectura asíncrona, multitenant y escalable

Ventajas

- Captura inteligente y rápida: Los usuarios pueden subir fotos o PDFs desde el móvil; el sistema extrae automáticamente los datos mediante IA y OCR de Microsoft AI Foundry.
- Automatización y eficiencia: Reduce el esfuerzo manual, minimiza errores y acelera los procesos de validación e integración con Fundanet y otros sistemas.
-  Seguridad y cumplimiento: Integración con Azure B2C y OpenID Connect, almacenamiento seguro de imágenes en Azure Blob Storage, y cumplimiento del RGPD y LOPDGDD.
- Escalabilidad y fiabilidad: Infraestructura basada en Azure que garantiza alta disponibilidad, balanceo de carga y procesamiento distribuido.

### Proyecto Nexus
Nexus es una plataforma integral diseñada para interconectar diferentes sistemas y plataformas utilizados en los Centros de Investigación facilitando la comunicación y garantizando una integración fluida. Su principal objetivo es facilitar la interoperabilidad entre aplicaciones, historiales clínicos, sistemas de análisis y dispositivos de monitorización, promoviendo una gestión eficiente de la información y mejorando la colaboración entre herramientas especializadas.  
Nexus es capaz de interaccionar con un gran número de aplicaciones.
* Fundanet – Ctms e IPharma  
* Historiales clínicos de la plataforma de SAP Argos 
* Visor de monitores SAP Argos. 
* OpenClinica 
* RedCap 
* Telemedicina Docline 
* E-CAP 
* SISCat 
* Fenix Erp 
* Tarjeta sanitaria SCS 
* Visor HCE SCS 
* Explotación de datos a través de PowerBi 
* Integración con dispositivos Apple Health y Android Google Fit SDK 

Tecnologías utilizadas:
- Mirth Connect versión Comercial
- Servicios de procesamientos de mensajes desarrollado con Net Core 9
- Azure Service Bus: Message broker (bus de mensajería) que actúa como middleware para enviar y recibir eventos JSON entre servicios. 
- Usa mensajes en formato Protobuf: Estos mensajes representan los eventos de dominio (por ejemplo, CreacionPaciente, NuevoEstudio, etc.) intercambiados entre servicios.
- Api Rest para exponer los endpoints de integración
- Utiliza Azure Functions para la el procesamiento de mensajes en la nube y su integración con Webhookds de clientes externos.
- Implementa el Patrón Outbox e Inbox: Que permite garantizar consistencia entre base de datos y mensajes enviados, registrando los eventos en una tabla “Outbox” y publicándolos de forma segura y asegurar idempotencia (Inbox, no procesar dos veces el mismo mensaje), guardando un registro de los mensajes ya consumidos.
- Aplicaciones para monitorizar y reintentar mensajes fallidos.
- Sistema de trazabilidad y auditoria 
- Resiliente ante fallos y cortes de Internet.

Características principales de Nexus: 
* Interoperabilidad: Conexión fluida entre múltiples sistemas y dispositivos. 
* Automatización y comunicación: Nexus permite el uso de múltiples protocolos de comunicación, mediante mensajería, servicios basados en eventos, API Rest y otros. 
* Uso de estándares internacionales: Garantiza la calidad y seguridad en el manejo de la información de salud como HL7, OMOP, etc. 
* Análisis avanzado: Integración con herramientas como Power BI para la visualización y toma de decisiones basada en datos. 

 Nexus garantiza la interoperabilidad a través de estándares reconocidos internacionalmente como: 
* API REST, para servicios web modernos y escalables 
* HL7 FHIR, estándar para el intercambio de información clínica y la interoperabilidad rápida y flexible de datos de salud 
* OMOP CDM (Observational Medical Outcomes Partnership Common Data Model), para la normalización de datos clínicos y su análisis a gran escala 
* Mensajería basada en API REST y Webhooks, facilitando notificaciones y actualizaciones en tiempo real. 
* Integración con SDKs de dispositivos de salud de Android y Apple, para el intercambio de datos de actividad y monitorización médica 




### Fundanet Suite - Lanzador de aplicaciónes

La Suite de aplicaciones trata por un lado de integrar un sistema de Autenticación único basado en B2C, de forma que cada usuario que acceda una vez autenticado tenga la opción de lanzar cualquier aplicación de la Suite a la que se tiene acceso

Una suite de aplicaciones es un conjunto de programas o aplicaciones de software que están diseñados para trabajar juntos y cumplir con una serie de funciones relacionadas, generalmente dentro de un mismo entorno o para un propósito común. Estas aplicaciones suelen estar integradas de manera que puedan compartir datos y funcionalidades, facilitando la colaboración y optimizando los flujos de trabajo.

La idea es tener una entrada común para cada usuario, de forma que este pueda acceder a cada uno de los programas. Esto conforma un punto de entrada único, el sistema además puede mostrar estadísticas y notificaciones, caducidad de las licencias, incidencias en las aplicaciones, estadísticas de uso, etc.

Esta aplicación esta desarrollada en Blazor Web Assembly y usa una autenticación Oauth con B2C, valida, los usuarios, las licencias y las aplicaciones disponible y las presenta para poder acceder a ellas con facilidad.





Fundanet CTMS

Fundanet CTMS es una plataforma pensada para gestionar ensayos clínicos (Clinical Trial Management System). Ofrece una solución integrada para los centros de investigación, sitios de ensayos clínicos, universidades y oficinas de transferencia tecnológica. 
El enfoque del software está en optimizar los procesos operativos, asegurar cumplimiento regulatorio y financiero, y facilitar la colaboración entre roles y departamentos dentro de un ensayo.

Ventajas:

* Mejora de la eficiencia operativa al centralizar y automatizar tareas. 
* Mejores capacidades analíticas y visibilidad del estado del ensayo. 
* Potencial para aumentar ingresos mediante facturación más precisa y evitar errores como doble facturación. 
* Capacidad de adaptación del sistema a las necesidades específicas del sitio de investigación. 
* Interoperabilidad con otros sistemas (EMR, EDC, facturación).

Arquitectura:

- Framework ABP basado en Asp Net Core y DDD
- Azure Sql Database
- Azure Key Vault
- Azure Blob Storage
- Integración B2B 

Características principales:

- Gestión de estudios clínicos: Control centralizado de datos, coordinación de tareas, gestión de sitios y participantes, generación de reportes y análisis. 
- Coordinador clínico.	Manejo de información de los participantes, cumplimiento regulatorio, recopilación de datos y reportes. 
- Gestión financiera. Gestión presupuestaria, asignación de recursos, contratos y subvenciones, procesos automáticos de facturación y prevención de doble facturación. 
- Colaboración y comunicación: Integración de roles (investigadores, coordinadores, finanzas) en una plataforma común para decisiones coordinadas. 
- Integraciones / interoperabilidad: APIs REST, uso de Azure Service Bus, conexión con sistemas de EMR (historias clínicas electrónicas), EDC, sistemas de facturación, etc. 


Fundanet IEC

Fundanet IEC es una plataforma especializada para la gestión de comités de ética en ensayos clínicos, estudios observacionales y proyectos biomédicos. Está diseñada para facilitar los procesos del comité de ética en entornos académicos, clínicos e institucionales. 
El objetivo es soportar la presentación, revisión, aprobación y seguimiento de proyectos de investigación desde el punto de vista ético, con eficiencia, transparencia y cumplimiento regulatorio

Ventajas:

* Eficiencia administrativa: al permitir presentaciones electrónicas y flujos automáticos, reduce papeleo y tiempos. 
* Mayor control y trazabilidad: con control de versiones y monitoreo continuo, se puede tener un registro claro del estado de cada proyecto. 
* Mejora en coordinación: al automatizar asignaciones y revisar progresos, los comités de ética pueden trabajar de forma más ordenada. 
* Flexibilidad del sistema: se puede ajustar según las reglas internas y procesos propios del comité de ética en cada institución. 
* Transparencia y cumplimiento regulatorio: al centralizar la documentación y controlar versiones, ayuda a reforzar estándares éticos y regulatorios.

Arquitectura:

- Framework ABP basado en Asp Net Core y DDD
- Azure Sql Database
- Azure Key Vault
- Azure Blob Storage
- Integración B2B 

Características principales:

- Envíos electrónicos:	Permite que los investigadores hagan presentaciones (submissions) mediante un portal seguro en línea, reduciendo la carga administrativa. 
- Flujo de revisión automatizado: Las tareas de revisión se asignan automáticamente, se monitorea el progreso de cada revisión, y el sistema ayuda a coordinar revisores y estados del proyecto. 
- Gestión documental integral: Ofrece un repositorio de documentos con control de versiones (version control) para manejar documentos como protocolos, enmiendas, reportes, consentimientos, etc. 
- Configuración: Posibilidad de adaptar parámetros y preferencias del sistema según las necesidades específicas del comité de ética (por ejemplo reglas internas, plazos, rutas de revisión). 
- Calendarios: Administración de calendarios para citas, reuniones del comité, plazos importantes y eventos relacionados con el comité de ética. 
- Gestión de datos: Herramientas para entrada de datos, revisión, verificación, asegurando la integridad de la información relacionada con las presentaciones. 
- Reportes: Generación de reportes personalizados para análisis del desempeño del comité, estados de proyectos, tiempos de revisión, estadísticas, etc. 
- Monitoreo: Seguimiento continuo del estado de los proyectos sometidos al comité, control de cumplimiento de protocolos, supervisión del ciclo de vida de las revisiones. 
- Integración: Capacidad de integrarse con otros sistemas y plataformas utilizados en investigación clínica, para intercambio de datos entre departamentos o sistemas externos.
